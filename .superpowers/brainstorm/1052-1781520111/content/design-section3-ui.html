Design — Section 3: Admin Feature Structure & UI

File layout, component breakdown, and AdminService responsibilities.

Feature Folder Structure

src/app/features/admin/
admin/
├── admin-dashboard.component.ts  ← root shell (nav + router-outlet)
├── components/
├── event-grid.component.ts  ← search, table, toggle, delete
├── event-edit-modal.component.ts  ← edit form + verification guard
└── magic-link-panel.component.ts  ← user lookup + manual send
└── admin.service.ts  ← all admin Supabase queries

AdminService API

admin.service.ts — methods
getAllEvents()  → all events across all hosts, with host email joined from profiles
getAllProfiles()  → all users (for magic link lookup)
toggleEventEnabled(id, is_enabled)  → flip is_enabled on event
updateEvent(id, fields)  → patch any event fields
deleteEvent(id)  → hard delete (cascade wipes guests + rsvps)
sendManualMagicLink(email)  → calls supabase.signInWithMagicLink() for target email
